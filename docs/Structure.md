# Project Structure

The project follows a modern, decoupled architecture with a Node.js/Express backend (`server.js`) and a Preact/Vite frontend (`src/`).

## Root Directory

*   **`server.js`**: The Express backend server. Handles the SQLite database, file uploads (`multer`), JWT authentication, and REST API routes.
*   **`data/`**: Stores backend persistent data.
    *   `database.sqlite`: The SQLite database containing `articles` and `auth` tables.
*   **`public/` & `assets/`**: Static files and user-uploaded media (`public/uploads/`).
*   **`archive/`**: Contains historical documentation, including old security and bug reports.
*   **`docs/`**: Current active documentation (Structure, Code, Functions).
*   **`dist/`**: The production build output generated by Vite.

## Frontend (`src/`)

The frontend is built with Preact and organized into logical, domain-specific directories:

### `src/components/`
Reusable UI components that are shared across different pages.
*   **`Layout.jsx`**: The main page wrapper containing the `Navbar`, main content area, and footer.
*   **`Navbar.jsx`**: Responsive navigation bar with cinematic hover animations, language toggle, and theme switch.
*   **`BackToTop.jsx`**: A utility button that appears on scroll to return to the top of the page.
*   **`MarkdownViewer.jsx`**: A secure wrapper component that parses markdown and sanitizes it against XSS using `DOMPurify`.

### `src/hooks/`
Custom Preact hooks leveraging `@preact/signals` for global, reactive state management.
*   **`useArticles.js`**: Manages fetching, creating, updating, and deleting articles from the API.
*   **`useAuth.js`**: Handles JWT token storage, login flows, and initial admin setup.
*   **`useLanguage.js`**: Manages i18n (English/Italian translations) and persists the choice in `localStorage`.
*   **`useTheme.js`**: Toggles between dark mode and light mode, reacting to system preferences and user overrides.

### `src/pages/`
Route-level components that represent full pages.
*   **`Home.jsx`**: The landing page with a cinematic splash screen loader, hero section, and recent news preview.
*   **`About.jsx`**: Details about the studio and the founders (Vinci & Nicolò).
*   **`Portfolio.jsx`**: A showcase for cinematic projects.
*   **`Articles.jsx`**: The main news feed / blog list.
*   **`ArticleDetail.jsx`**: The individual article reading view, rendering sanitized markdown.
*   **`Contact.jsx`**: A standalone contact page featuring interactive email and Instagram links.
*   **`Admin.jsx`**: A secured dashboard for the site owner to manage content, upload images, and write markdown articles.

### Core Entry Points
*   **`main.jsx`**: The application entry point that mounts Preact to the DOM.
*   **`app.jsx`**: The primary router component that handles the custom `InitialLoader` and maps paths to `src/pages/` components.
*   **`index.css`**: Global stylesheet initializing Tailwind CSS directives and custom properties.
